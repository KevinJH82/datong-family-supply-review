# 大同家庭补货助手 MVP

这是一个重线上、轻线下的 MVP 原型，用来验证“大同家庭补货清单服务”是否能从内容平台承接用户，并通过小程序式测评和私聊成交完成首单转化。

## 当前实现

- 用户端首页：项目定位、套餐入口、运营节奏、信任说明
- 家庭测评：家庭人数、孩子年龄、预算、忌口、常购品类、冷链需求、收货区域、微信
- 推荐结果：按成本表、预算和家庭结构生成补货包、品类建议、履约提醒、预估毛利
- 成本表：录入采购城市、物流摊销、服务费率、损耗预留、冷链包装费、SKU 当天采购价
- 意向单：提交后写入浏览器本地存储
- 管理端：查看线索、筛选状态、导出 CSV、清空演示数据
- 内容素材：内置小红书/抖音内容选题，便于持续获客

## 使用方式

直接用浏览器打开 `index.html` 即可。

也可以在目录内启动本地服务：

```bash
python3 -m http.server 8088
```

然后访问：

```text
http://localhost:8088
```

## 迁移到微信小程序时的边界

当前版本刻意不做在线支付、实时库存、购物车和多团长分销。迁移到微信小程序时，建议保留同样的 MVP 边界：

- 首页
- 测评页
- 推荐结果页
- 意向单提交
- 管理后台或运营端表格

等 30 天验证指标达标后，再考虑微信支付、订单状态、会员权益、复购提醒。

## 30 天验证指标

- 小程序/页面测评提交：100 份
- 首单转化：20-30 单
- 复购或会员意向：5 个
- 测评到首单转化率：15% 以上
- 首单客单价：399 元以上

## 真实落地流程

1. 采购前一天打开“成本表”，把每个 SKU 改成采购城市山姆 App/门店的当天实付价。
2. 录入单单物流摊销、冷链包装费、损耗/售后预留和服务费率。
3. 用户提交测评后，系统会生成预算包、商品成本小计、总成本、预估毛利和毛利率。
4. 私聊时只承诺“按预算做最终确认”，不要承诺固定 SKU；缺货时按同品类同预算替换。
5. 如果某个预算包预估毛利率低于 12%，不要成交，改为提高预算、减少冷链或调整 SKU。

## 价格口径

`399/699/999` 是预算档，不是固定商品组合。当前内置 SKU 价格为公开参考价，只用于试算。正式对外报价前，必须替换为实际采购价。

## 小红书每日运营机制

每天围绕“小红书内容 -> 测评页 -> 私聊确认 -> 成本表试算 -> 首单/复购”这条链路运营。

每日需要准备的信息：

- 今天主推商品或预算包
- 是否有真实商品图、到货图、分拣图、小票图
- 昨日数据：曝光、点赞、收藏、评论、私信、测评提交数、下单数

每日产出内容：

- 当日选题
- 3-5 个标题
- 小红书正文
- 图片脚本和封面文案
- 评论区引导
- 私信承接话术
- 次日优化建议

建议日更节奏：

- 周一：本周补货清单
- 周二：山姆商品避坑
- 周三：有娃家庭零食/早餐
- 周四：预算包案例
- 周五：周末到货预告
- 周六：真实到货/分拣图
- 周日：一周复盘/下周预告

## 合规提醒

- 不宣称山姆官方授权
- 不使用“官方价”“官方代购”等误导表达
- 涉及食品销售前，确认营业执照、食品经营许可或仅销售预包装食品备案要求
- 保留采购小票、到货照片、售后记录
